# Daily Synthesis: 2026-04-10

The strongest signal today is that the agent stack is being forced to grow up in three places at once: runtime contracts, realistic evaluation, and the actual substrate of control. Interoperable event surfaces matter more than another planner abstraction. Live-site benchmarks matter more than sandbox win rates. And sovereignty depends less on vague local-first branding than on whether the user owns the shared state layer and the runtime can survive hostile instruction channels.

## Top findings

### 1. PydanticAI v1.79.0 pushes agent runtimes toward interoperable UI and capability contracts
- Core source: https://github.com/pydantic/pydantic-ai/releases/tag/v1.79.0
- Why it matters: AG-UI support, capability `apply()`, and explicit async HTTP client lifecycle management all move agent systems toward cleaner runtime contracts instead of framework-specific glue.
- Implementability score: 0.91

### 2. ClawBench forces agent evaluation onto live websites instead of fake sandboxes
- Core source: https://arxiv.org/abs/2604.08523v1
- Why it matters: 153 tasks across 144 live platforms with final-request interception is a much more honest test of whether web agents can do ordinary work safely.
- Implementability score: 0.78

### 3. PIArena turns prompt injection defense into a real comparative evaluation problem
- Core source: https://arxiv.org/abs/2604.08499v1
- Durable topic: ../Strategy/runtime-governance/runtime-governance.md
- Why it matters: adaptive attacks and cross-benchmark comparisons expose how weak many current prompt-injection defenses really are.
- Implementability score: 0.76

### 4. PSI argues that shared state is the missing sovereignty layer for personal agents
- Core source: https://arxiv.org/abs/2604.08529v1
- Durable topic: ../Strategy/shared-state-agents/shared-state-agents.md
- Why it matters: persistent shared state plus governed write-back paths is a much better foundation for personal agents than transcript-centric memory or isolated generated apps.
- Implementability score: 0.57

## What is implementable now

The highest-leverage moves right now are:
- standardize agent runtime and UI events around explicit contracts rather than bespoke glue
- build live-site eval harnesses with request interception so agents can be tested realistically without causing side effects
- evaluate prompt injection defenses across multiple tasks and adaptive attack strategies, not one curated benchmark
- move personal-agent architecture toward typed shared artifacts and governed write-back rather than chat-only state

## Highest and lowest implementability today
- Highest: PydanticAI-style interoperable runtime contracts at 0.91
- Lowest: PSI-style shared-state personal-agent architectures at 0.57

## What I scanned but did not elevate
- Microsoft Agent Framework 1.0.1 is useful signal, especially restricted checkpoint deserialization and new retrieval/memory samples, but it reads more like hardening of an already-covered runtime trend than a new category shift.
- Waypoint-1.5 is a strong local world-model update for everyday GPUs, but it matters more for simulation substrate than for the current agentic stack map.
- Act Wisely is a good paper on meta-cognitive tool arbitration in multimodal agents, but today's bigger practical move is still evaluation and systems discipline rather than another training trick.

## Opinionated read

The field is slowly being dragged away from demo-first thinking. Good. The useful teams will be the ones that make their runtimes explicit, their evaluations honest, and their state layer ownable.
