## What is implementable now

The clearest moves you can make now are:
- turn agent workflows into reusable skill packs, harnesses, and artifact-first workspaces
- promote distilled validation routines and debugging heuristics into durable memory with provenance
- score agents on execution quality, parameter accuracy, and recovery behavior in live environments
- treat checkpoint restore, sandbox boundaries, and instruction hierarchy as part of runtime design
- evaluate platforms on durable execution and idle-cost behavior, not just model access

## Highest and lowest implementability this week
- Highest: portable procedure stacks at 0.94
- Lowest: personal-agent trust and consent calibration at 0.63

## Clear model of the stack right now
- **Implementable now:** workflow packs, artifact-first shared state, explicit memory promotion, bounded sandboxes, checkpoint threat modeling, execution-aware evals, kernel-plus-client agent products
- **Architecture-heavy but tractable:** runtime operating systems with clean migration paths, multi-agent authority inheritance, cross-runtime state portability, deeply consent-aware personal agents
- **Still mostly conceptual:** any story that assumes general-purpose autonomous agents will become trustworthy without explicit governance surfaces, provenance, and runtime boundaries

## Opinionated read

The winning agent stack no longer looks like “better prompts plus more tools.” It looks like a controlled runtime with portable procedure, memory that can actually transfer, benchmarks that watch real execution, and infrastructure that makes authority and failure legible.

## Strategic question for next week

If the runtime is becoming the true control plane, which layer should you own yourself: the memory system, the workflow package format, the evaluation harness, or the runtime substrate itself?
