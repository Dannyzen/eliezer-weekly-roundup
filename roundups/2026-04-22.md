# Daily Scan: 2026-04-22

This week’s strongest signal is that serious agent frameworks are making the execution substrate first-class. OpenAI's sandbox-native turn packages persistent isolated workspaces, manifests, snapshots, resume support, and explicit backend choice into the worker itself. Around that core shift, delegation is moving from fixed role labels to context-conditioned evidence, computer-use reliability is being redefined around repeatability, local-first claims are getting tested by subtask and backend instead of ideology, and high-autonomy security tools are starting to ship with containment baked into the runtime.

## Deep Dive Wednesday winner

**Winner:** OpenAI Agents SDK's sandbox-native turn → [Sandbox-Native Agent Workers](../AgenticAI/sandbox-native-agent-workers/sandbox-native-agent-workers.md)

Why it won the week:
- it makes execution substrate design a shipped SDK primitive instead of a sidecar concern
- it operationalizes the repo's recent themes around file-as-bus workspaces, harness architecture, resumability, and containment
- it is architecturally bigger than the week's strong delegation, evaluation, and containment papers because it changes what the default worker should be
- it is implementable now with standard engineering effort

- use local or Docker sandbox clients as explicit execution backends

- keep worker memory and mounts explicit so state stays inspectable
- separate thin orchestration from bounded execution workers

## What is implementable now

The clearest moves you can make now are:
- model long-running work as bounded workers with manifests instead of free-floating shell loops
- snapshot and resume agent execution instead of forcing every meaningful task into one session
- route multi-agent work using per-context performance histories instead of one static specialist score
- evaluate browser and desktop agents under repeated execution before trusting a pass rate
- split local-first workloads by task shape so structured extraction stays local while long-context correction escalates selectively
- treat sandbox topology and writable boundaries as part of the product whenever an agent can execute high-risk actions

## Opinionated read

The field is getting less theatrical and more infrastructural. That is good. The next edge will come from teams that can say which worker should run this job, where it runs, what state it preserves, how often its behavior repeats, and exactly what boundary keeps a high-autonomy agent from turning into an uncontrolled shell.
