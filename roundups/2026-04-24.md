# Daily Scan: 2026-04-24

Today's strongest signal is context economy. DeepSeek-V4, Tool Attention, Claude Context, and StructMem all point at the same agent-stack constraint from different directions: more context is not enough. Tool schemas need gating, codebases need retrieval, memory needs structure, and long-context models need KV-cache economics that survive real agent loops. On the strategy side, Google's TPU 8t/8i split shows cloud infrastructure vendors are now designing separate paths for training and low-latency agentic serving.

## What is implementable now

The clearest moves you can make now are:
- add token-category telemetry to every serious agent loop: tools, memory, retrieved code, previous turns, and user input
- gate MCP tools through summaries, embeddings, state, and access policy before injecting full schemas
- use semantic code-search MCPs such as `zilliztech/claude-context` instead of loading entire repositories into prompts
- structure long-term memory around events, timestamps, provenance, and relationships, then run consolidation off the critical path
- benchmark long-context agent sessions by KV-cache growth and latency, not just maximum accepted context length
- separate training, post-training, and serving requirements when evaluating cloud infrastructure for agents

## Opinionated read

The stack is moving from context abundance to context accounting. The teams that win will not be the ones that paste the most text into the biggest window. They will be the ones that know which tools, memories, code, and traces deserve to be active at each step.
