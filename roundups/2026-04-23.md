# Daily Scan: 2026-04-23

Today's strongest signal is that the agent loop itself is becoming the product surface. OpenAI's workspace agents and WebSocket-backed Responses API point to the same architecture from two angles: long-lived runs with reusable state, approval gates, and transport that does not rebuild history after every tool call. The evaluation correction is just as important: SWE-chat shows that coding-agent quality looks very different once you measure real user traces, code survival, interruptions, and security regressions. On the strategy side, sovereignty is becoming more concrete through local privacy filters that redact before cloud escalation and local multimodal stacks that can run on Jetson-class hardware when the tool surface stays tiny.

## What is implementable now

The clearest moves you can make now are:
- treat multi-step agent work as a long-lived run with reusable state instead of rebuilding context turn by turn
- add approval checkpoints, analytics, and replayable run logs to any shared or scheduled agent
- measure coding agents by code survival, interruption, and security quality, not only by benchmark pass rates
- place a local privacy filter in front of cloud escalation, retrieval ingestion, and log retention
- prototype narrow local multimodal assistants with one or two explicit tools on OpenAI-compatible local endpoints

## Opinionated read

The stack is getting more operational. Better agents will come from teams that treat runs as sessions, evaluation as field telemetry, privacy as a local boundary stage, and local multimodal autonomy as a narrow tool problem instead of a general-intelligence fantasy.
