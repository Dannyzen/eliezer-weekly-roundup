## Signals

- **Highest implementability:** MCP semantic gateway governance at 0.76. The pieces exist today: gateways, OAuth/OIDC, RBAC, OPA/Cedar, OpenTelemetry, Prometheus, and adversarial test harnesses.
- **Lowest implementability:** knowledge-state orchestration at 0.58. The pattern is clear, but designing state schemas and evaluator loops that improve synthesis without adding bureaucracy is still hard.
- **Most actionable engineering move:** start versioning harness components and attach predicted-effect notes to harness changes.
- **Most important governance move:** put a real gateway in front of MCP tools before adding more autonomous workflows.

## Actionable Insights

1. Treat harness edits like software changes: version, test, replay, and verify predicted outcomes.
2. Stop using one transcript as the only memory for long-horizon research; store claims, evidence, assumptions, and artifacts explicitly.
3. Put identity and tool-level RBAC in front of MCP servers before scaling agent access.
4. Add routing logs that state why a workflow used a managed cloud model, a local/open model, or a hybrid path.
5. Use GitHub Trending as a demand signal, not a source of truth: developer attention is flowing toward agentic terminals, skills, harnesses, and code-graph context tools.
